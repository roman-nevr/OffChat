Написать оффлайн-чат с ботом.
Приложение должно представлять из себя экран чата с полем для ввода сообщения.
Дизайн приложения (графика) не имеет большого значения.

При первом заходе в приложение в чат автоматически отсылаются координаты местоположения пользователя.
Бот рандомно отправляет сообщения раз в 2-3 секунды.
Также должна быть возможность отправить текущую геолокацию и картинку (выбрать из галереи или снять при помощи камеры).

Все сообщения должны храниться в локальной базе данных. Для обновления списка сообщений выполнить подписку на БД. БД это лишь локальная копия сообщений с сервера (можно считать что сейчас он находится в разработке), поэтому сделать методы-заглушки для отправки сообщений на сервер, которые в будущем можно заменить на рабочий код, как только backend заработает.

При свернутом приложении должны отображаться нотификации от бота в статус баре.
По клику по нотификации открывать экран чата.
Если убить приложение нотификации должны по прежнему приходить, в приложении должна быть возможность отключить эту функцию.

При выполнении задания уделить внимание архитектуре приложения, чистоте кода.
Основной критерий выполненного ТЗ - отсутствие кода, который бы хотелось улучшить. Пишите так как вы бы писали для продакшена.
Тесты приветствуется.
Приветствуется использование стилей для оформления UI элементов, вынесение стилей, возможно использование наследования (где это уместно). То же касается других ресурсов (строк, цветов, числовых констант, атрибутов)»

Хотелось бы увидеть демонстрацию работы с rxjava в тестовом задании.
